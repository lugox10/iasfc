
# iasfc
# Se creó el backend con la estructura de programación hexagonal.
# Se creó la base de datos en H2 por el momento.
# Se crearon los paquetes playermodels, playerRepository, playerService, playerController.
# En la clase Player del paquete playermodel se registrarán los datos del jugador: nombre, apellido, posición de juego, email y contraseña.
# Como principio del backend de la API REST (iasfc), el objetivo es lograr la inscripción de los jugadores y generar un administrador definido.
# Se creó la base de datos con XAMPP y phpMyAdmin, y se implementaron las tablas: player, rol, user_rol.
# Se está implementando Spring Security.
# Se utilizó Thymeleaf y HTML para la vista del formulario.
